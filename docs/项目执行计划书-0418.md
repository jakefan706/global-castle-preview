# Global Castle 网站项目执行计划书

> **公司**：Global Castle Industrial Co., Ltd.
> **项目**：B2B 能力展示官网（全新开发）
> **技术方案**：Next.js 16 + Payload CMS v3 + PostgreSQL
> **更新日期**：2026-04-18
> **版本**：v5（能力展示主导，分类与 CMS 结构统一）

---

## 一、项目背景

Global Castle 是一家位于杭州的外贸公司，主营杯壶类产品出口（不锈钢保温杯、铝瓶、塑料杯、陶瓷杯、玻璃杯等）。目标客户为欧美礼品商、批发商、咖啡连锁店和品牌商（纯 B2B）。

现有网站 cnurbaneco.com 设计过时、信息架构混乱、缺乏信任背书。曾尝试 Webflow 重建，但对零基础用户学习成本过高，尤其在多字段筛选和 CMS 配置方面。

现决定采用 **AI 辅助编码（Vibecoding）** 方案，使用 Next.js + Payload CMS 技术栈，实现完全自主可控的网站。

## 二、技术架构

### 2.1 技术栈选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端框架 | Next.js 16 (App Router) | SSG 静态生成，SEO 友好，性能优异 |

**导航与分类的关系**：Products 下拉菜单中的 11 个分类是快捷入口，点击后跳转到该分类的产品展示页。网站不提供筛选功能，分类承担 CMS 内容归类、导航深链和产品语义组织的作用。

### 3.3 Categories CMS 字段规范

`Categories` collection 统一作为产品分类主数据，字段定义如下：

| 字段 | 类型 | 用途 |
|------|------|------|
| `name` | text | 分类名称，后台标题与前台展示名 |
| `slug` | text / unique | 分类稳定标识，用于导航深链与 CMS 内容映射 |
| `description` | text | 分类简短说明，可用于列表页或卡片文案 |
| `image` | upload | 分类封面图或代表视觉 |
| `sortOrder` | number | 控制后台与前台的显示顺序 |
| `featuredOnHomepage` | checkbox | 标记是否作为首页主力分类展示 |

## 四、页面详细设计

### 4.1 首页 (Home)

参考：YourCustomBottle + YewayDrinkware + sensialu.com

**建议先用 v0.dev 生成 UI 原型确认方向后再编码。**

| 区块 | 内容 |
|------|------|
| **Hero** | 全宽大图 Banner（桌面端 1920×1080，移动端 768×1024）+ 大标题 + 副标题 + 双 CTA（"View Products" + "Get Quote"） |
| **核心优势** | 3-4 个图标卡片：20 年经验 / 全球认证 / 快速交货 / 定制服务 |
| **产品分类入口** | 4-6 个大卡片，点击进入对应分类的产品展示页，不再额外显示 11 个分类 chips |
| **核心数据** | 动画数字展示：成立年份、年产能、出口国家数、认证数量 |
| **信任背书** | 认证徽章（GRS/FDA/ISO 等）+ 合作品牌 Logo 墙（如有） |
| **快速询盘** | 简化表单：姓名、邮箱、公司、留言 |

**Hero Banner 规格**：

| 项目 | 要求 |
|------|------|
| 格式 | JPG 或 WebP |
| 桌面尺寸 | 1920×1080px（16:9） |
| 移动端尺寸 | 768×1024px（3:4，竖版裁切） |
| 文件大小 | 桌面版 ≤ 400KB，移动版 ≤ 200KB |
| 内容建议 | 工厂实拍或产品铺陈，主体居中或偏右，左侧留白给文字叠加 |

### 4.2 产品列表页 (Products)

**定位调整**：网站以能力展示为主，产品展示辅助。产品页面不做大而全的目录式展示，而是精选每个分类的代表性产品（10-15 款），重点展示工艺能力、材质多样性和定制能力。

**页面结构**：

- **顶部**：页面标题 + 当前分类说明
- **分类到达方式**：通过 Header Products 下拉的分类深链进入，不做站内筛选或 Tab 切换
- **产品网格**：每个分类展示 10-15 款精选产品，无分页
- **底部 CTA**：「Have a specific product in mind? Contact us for the full catalog.」→ 引导询盘

**产品卡片**：主图 + 产品名 + 型号 + 材质标签 + 「View Details」按钮

**网格响应式**：桌面 4 列 / 平板 3 列 / 手机 2 列

**无筛选功能**：不提供筛选面板、筛选 chips、筛选结果切换等交互。如需查找特定产品，引导联系销售。

> **决策背景（2026-04-18）**：公司定位为能力展示型官网，产品数量精选而非大而全。分类保留用于 CMS 管理、导航深链和产品语义组织，但不作为前台筛选系统。

### 4.3 产品详情页 (Product Detail)

**建议先用 v0.dev 生成 UI 原型确认方向后再编码。**

参考：https://rticoutdoors.com/Happy-Hour-Pint-Tumbler （布局和视觉叙事风格）

借鉴其核心设计模式，适配 B2B 场景：

**Section 1 — 产品总览（首屏）**
- 左侧：产品图片画廊（主图 + 缩略图切换，支持多角度浏览）
- 右侧：产品名称、型号、核心卖点（3-5 个图标+短文字）、基础规格（容量/材质/重量）、认证标识、**"Send Inquiry" 按钮**

**Section 2 — 特性展示（图标卡片区）**
- 3-4 个特性卡片（图标 + 标题 + 简短描述）
- 例如：双层真空隔热 / BPA Free / 可定制 Logo / 通过 FDA 检测

**Section 3 — 视觉叙事段落（交替布局）**
- 参考 rticoutdoors 的交替「图片/视频 + 文字」布局
- 每段讲一个主题：制造品质 / 定制工艺 / 耐久性 / 使用场景
- 大量留白，呼吸感强，显得高端
- 内容来自产品 description 字段（Rich Text，支持图文混排）

**Section 4 — 技术规格**
- 结构化规格表（来自 specifications 字段）
- 清晰的表格或列表形式

**Section 5 — 相关产品**
- 同分类的其他产品推荐（卡片横排）

### 4.4 Solutions（定制解决方案）

参考：https://yourcustombottle.com/stainless-steel-drinkware-custom-solutions/

**静态页面**，内容直接写在代码中（变动频率低，需要改时用 AI 改代码即可，无需 CMS 管理）。

整体为一个长页面，包含 5 个解决方案模块：

#### Branding
- 各印花工艺对比介绍：UV Printing / Silkscreen Printing / Heat Transfer Printing / Laser Engraving
- 每种工艺：说明文字 + 效果图 + 适用场景 + 优缺点
- 展示不同印花工艺的适用场景与优势对比，不提供交互式筛选

#### Surface Finishing
- 各种喷涂工艺展示及特点介绍
- 效果对比图

#### Custom Design & Mold
- 设计能力和模具开发能力展示
- 开发流程（构思 → 2D 设计 → 3D 建模 → 开模 → 样品）

#### Custom Color
- Pantone 配色能力
- 颜色定制流程

#### Packaging
- 包装方案展示（白盒、彩盒、礼盒、定制包装）
- 各方案效果图

### 4.5 Resources（资源中心）

简化为 2 个板块：

| 子板块 | 内容 | 说明 |
|--------|------|------|
| **Catalog Downloads** | 历年广告单 PDF | 不在页面直接展示，仅持有链接的人可访问下载 |
| **Certifications** | 认证证书展示 | 名称 + Logo + 说明 + 证书图片，从 CMS 管理 |

原 Manufacturing Methods、QC & Testing、Product Knowledge 的文章内容统一归入 Blog 系统，Resources 页面提供指向 Blog 相关分类的链接入口。

### 4.6 Blog（统一文章系统）

- 图文文章列表（卡片网格：封面图 + 标题 + 摘要 + 日期）
- **按发布日期倒序**，最新在前
- 不需要分类筛选功能
- 文章详情页：正文（Rich Text）+ 分享按钮 + 相关文章推荐

**文章分类**（统一管理，替代原来分散的 Resources 文章和 Blog）：
- **Industry Guide** — 加工知识、产品知识、材质对比、选购指南（原 Manufacturing Methods + Product Knowledge）
- **Quality & Testing** — 检测方法、认证说明、第三方测试报告（原 QC & Testing）
- **Company News** — 公司动态、展会信息、新品发布

### 4.7 About Us

- 公司图文介绍
- 发展历程时间轴
- 核心数据可视化（成立年份、产能、出口国家数、认证数）
- 工厂/办公室照片

### 4.8 Contact

参考：https://yourcustombottle.com/contact-us/ （简洁风格）

- 简短引导文字 + 24 小时回复承诺
- **询盘表单**：姓名、邮箱、公司、国家、感兴趣的产品类别、留言
- 联系方式：邮箱、电话、WhatsApp

## 七、UI 设计流程

**首页**和**产品详情页**是全站设计要求最高的两个页面，建议在编码前先确认视觉方向。

### 推荐工作流

```
1. 用 v0.dev 输入描述 → 生成 UI 原型（React + Tailwind 代码）
2. 截图/预览 → 确认整体方向和布局
3. 如需调整 → 修改描述再生成 / 手动微调
4. 方向确认后 → 在项目中正式编码实现
```

### 需要做设计稿的页面

| 页面 | 是否需要设计稿 | 说明 |
|------|---------------|------|
| **首页** | 需要 | 设计要求高，Hero 视频 + 多区块布局 |
| **产品详情页** | 需要 | 参考 rticoutdoors 的叙事式布局，需确认风格 |
| 产品列表页 | 不需要 | 分类落地页 + 产品网格，结构简单，AI 直接生成 |
| Solutions | 不需要 | 参考 yourcustombottle 直接实现 |
| Blog / About / Contact / Resources | 不需要 | 标准页面，AI 直接生成 |

### 其他可选工具

| 工具 | 用途 |
|------|------|
| **v0.dev** (Vercel) | 首选，输出 React + Tailwind 代码，和技术栈完全兼容 |
| **Google Stitch** | 视觉风格探索 |
| **Figma** | 精确设计（学习成本较高，非必须） |

## 九、管理后台使用说明

团队成员通过浏览器访问 `域名/admin` 即可管理所有内容。

### 9.1 添加新产品

1. 登录管理后台 → 点击 **Products** → **Create New**
2. 填写产品名称、编号
3. 上传主图和多角度图片（拖拽上传，系统自动生成缩略图和 WebP）
4. 下拉选择分类、材质、容量、应用场景、价位
5. 填写产品描述（富文本编辑器，支持插入图片，用于详情页叙事段落）
6. 填写规格参数
7. 按需勾选 "Featured" 或 "New Arrival"
8. 点击 **Publish** → 产品页面自动生成

### 9.2 发布博客 / 资源文章

1. 点击 **Blog Posts** → **Create New**
2. 填写标题、摘要、上传封面图
3. 在富文本编辑器中编写正文
4. 选择分类（Industry Guide / Quality & Testing / Company News）
5. 设置发布日期 → **Publish**

### 9.3 管理认证

1. 点击 **Certifications** → **Create New**
2. 填写名称、上传 Logo、写简介、上传证书图片（可选）
3. **Publish** → 自动显示在 Resources 页面和首页信任背书区

### 9.4 查看询盘

1. 点击 **Inquiry Submissions**
2. 查看所有客户提交的询盘（同时会有邮件通知到销售邮箱）
