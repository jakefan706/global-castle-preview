# Global Castle 网站动画设计规范

> 本文档定义全站动画系统，作为开发实施的参考标准。
> 原则：**高级但克制** — 每个动画都有明确目的，拒绝为动画而动画。

---

## 一、设计原则

1. **有目的** — 每个动画必须服务于「引导注意力」「提供反馈」「建立空间关系」三者之一，纯装饰性动画不做
2. **统一节奏** — 全站使用 4 级标准时长，不出现随意的 duration 值
3. **性能优先** — 只动画 `opacity` 和 `transform`（GPU 加速），不动画 `width/height/margin/top` 等触发重排的属性

---

## 二、技术方案

| 项目 | 选型 | 说明 |
|------|------|------|
| 过渡效果 | CSS `transition` | 按钮 hover、颜色变化、下拉菜单等 |
| 入场动画 | Tailwind 自定义 `@keyframes` | 滚动揭示、文字淡入、卡片入场 |
| 滚动触发 | Intersection Observer API | 原生 JS，不引入第三方库 |
| 数字滚动 | 轻量自定义 hook（~20 行） | 首页数据展示 |
| 图片画廊 | Embla Carousel | 产品详情页，轻量开源 |
| 动画库 | **不使用** Framer Motion / GSAP | 保持 bundle 轻量 |

### Tailwind 自定义 keyframes 配置

```css
/* 在 Tailwind CSS v4 的 @theme 中定义 */
@theme {
  --animate-fade-in: fade-in 0.5s ease-out forwards;
  --animate-fade-in-slow: fade-in 0.8s ease-out forwards;
  --animate-fade-up: fade-up 0.6s ease-out forwards;
  --animate-fade-up-slow: fade-up 0.8s ease-out forwards;
  --animate-fade-in-left: fade-in-left 0.8s ease-out forwards;
  --animate-scale-in: scale-in 0.3s ease-out forwards;
  --animate-count-up: count-up 2s ease-out forwards;
}

@keyframes fade-in {
  from { opacity: 0; }
  to { opacity: 1; }
}

@keyframes fade-up {
  from { opacity: 0; transform: translateY(24px); }
  to { opacity: 1; transform: translateY(0); }
}

@keyframes fade-in-left {
  from { opacity: 0; transform: translateX(-30px); }
  to { opacity: 1; transform: translateX(0); }
}

@keyframes scale-in {
  from { opacity: 0; transform: scale(0.95); }
  to { opacity: 1; transform: scale(1); }
}
```

## 三、计时体系

全站只使用以下 4 个标准时长，不允许出现其他值：

| 级别 | 时长 | 缓动函数 | 用途 |
|------|------|---------|------|
| **Instant** | `150ms` | `ease` | 颜色变化、复选框 toggle、焦点状态 |
| **Fast** | `300ms` | `ease` | 按钮 hover、链接下划线、下拉菜单展开 |
| **Normal** | `500ms` | `ease-out` | 图片切换、卡片 hover 效果、模块切换 |
| **Slow** | `800ms` | `ease-out` | 滚动揭示入场、叙事段落淡入、Hero 文字入场 |

特殊：首页数字滚动使用 `2000ms ease-out`（唯一的例外）。

## 四、全局动画（所有页面共享）

### 4.1 导航栏

| 动画 | 触发 | 效果 | 时长 |
|------|------|------|------|
| **Sticky 出现** | 向下滚动超过 Hero 区域 | 导航栏从顶部滑入 + 背景模糊 `backdrop-blur` + 阴影 | Fast 300ms |
| **下拉菜单展开** | hover 导航项（桌面）/ 点击（移动端） | opacity 0→1 + translateY(-8px→0) | Fast 300ms |
| **下拉菜单收起** | 离开导航项 | opacity 1→0 + translateY(0→-8px) | Fast 300ms |
| **汉堡菜单（移动端）** | 点击汉堡图标 | 侧边栏从右侧 translateX(100%→0) 滑入 + 背景遮罩 opacity 0→0.5 | Normal 500ms |
| **导航链接 hover** | hover | 文字颜色渐变到主色 `#67c0bf` | Instant 150ms |

### 4.2 按钮效果（重点参考 sensialu.com）

#### 主按钮（"Get a Quote" / "Send Inquiry" / "View Products"）

```
默认：背景 #67c0bf，白色文字，圆角
Hover：背景色加深 → #5ab0af，同时 box-shadow 从内部扩展
       box-shadow: inset 0 0 0 0 rgba(0,0,0,0.1) → inset 0 0 0 40px rgba(0,0,0,0.08)
Active：scale(0.97)
过渡：transition: all 300ms ease
```

效果说明：hover 时有一个从中心向外扩散的微妙阴影填充效果（学自 sensialu.com 的 box-shadow inset 技巧），配合颜色加深，手感高级。Active 时轻微缩小提供点击反馈。

#### 次按钮（边框样式，"View Details" / "Learn More"）

```
默认：透明背景，#67c0bf 边框 + 文字
Hover：背景填充 #67c0bf，文字变白
       background-color: transparent → #67c0bf
       color: #67c0bf → #ffffff
Active：scale(0.97)
过渡：transition: all 300ms ease
```

#### 文字按钮 / 链接

```
默认：文字颜色 #333，无下划线
Hover：文字颜色 → #67c0bf，::after 伪元素下划线从 opacity 0 → 1
       下划线颜色 #67c0bf，高度 2px，宽度 100%
过渡：color transition 150ms，::after transition 300ms
```

效果说明：hover 时颜色先变（150ms），然后下划线平滑显现（300ms），有先后节奏感（学自 rticoutdoors）。

### 4.3 页脚图标 hover（学自 sensialu.com）

```
默认：圆形图标，#67c0bf 边框，透明背景
Hover：box-shadow: inset 0 0 0 0 #67c0bf → inset 0 0 0 30px #67c0bf
       图标颜色 → 白色
过渡：transition: all 500ms ease
```

效果说明：从圆心向外用 box-shadow inset 填满整个圆形，非常高级的填充动画。这是 sensialu.com 最突出的交互效果之一。

### 4.4 滚动揭示（Scroll Reveal）

通用的「滚动到可视区域时元素入场」效果，用 Intersection Observer 触发。

| 效果名 | 动画 | 适用场景 |
|--------|------|---------|
| **fade-up** | opacity 0→1 + translateY(24px→0), 800ms | 默认入场效果，大多数内容块 |
| **fade-in** | opacity 0→1, 500ms | 图片、徽章等不需要位移的元素 |
| **fade-in-left** | opacity 0→1 + translateX(-30px→0), 800ms | 强调型标题，叙事段落交替入场的左侧元素 |

**触发规则**：
- 元素进入视口 20% 时触发（`threshold: 0.2`）
- 每个元素只触发一次（触发后移除 observer）
- 同组元素交错入场：每个元素延迟 `index * 100ms`（如 4 张卡片依次出现）

**移动端调整**：位移距离减半（24px → 12px），避免在小屏上感觉位移过大。

### 4.5 图片懒加载淡入

```
图片加载完成前：opacity 0
加载完成后：opacity 0 → 1, 500ms ease-out
```

通过 Next.js Image 组件的 `onLoad` 回调触发。

### 4.6 WhatsApp 浮动按钮

```
页面加载 1.5 秒后：从右下角 scale(0) + opacity(0) → scale(1) + opacity(1), 500ms ease-out
Hover：scale(1.1), 300ms
```

不做呼吸灯/脉冲等过度效果，保持克制。

## 五、页面级动画

### 5.1 首页 (/)

| 元素 | 触发 | 效果 | 时长 |
|------|------|------|------|
| **Hero 视频覆层文字** | 页面加载 | 大标题 fade-in-left 800ms，副标题 fade-in 500ms（延迟 300ms），CTA 按钮 fade-up 500ms（延迟 600ms） | 总计 ~1.1s |
| **核心优势卡片**（4 张） | 滚动揭示 | fade-up 交错入场，每张延迟 100ms | 800ms + stagger |
| **产品分类入口卡片** | hover | 图片 scale(1→1.05) + 叠加层 opacity 渐显 | Normal 500ms |
| **产品分类入口卡片** | 滚动揭示 | fade-up 交错入场 | 800ms + stagger |
| **核心数据数字** | 滚动揭示 | 数字从 0 滚动到目标值（如 0 → 20+） | 2000ms ease-out |
| **认证徽章行** | 滚动揭示 | fade-in（无位移，简单淡入） | 500ms |
| **快速询盘区块** | 滚动揭示 | 整块 fade-up | 800ms |

**Hero Banner 本身**：静态大图，无动画。文字叠加层动画如下。移动端使用竖版裁切图（768×1024）。

### 5.2 产品列表页 (/products)

| 元素 | 触发 | 效果 | 时长 |
|------|------|------|------|
| **产品卡片** | hover | 整张卡片 translateY(0→-4px) + box-shadow 加深 | Fast 300ms |
| **产品卡片图片** | hover | scale(1→1.05)，overflow hidden 裁切 | Normal 500ms |
| **分类标题区** | 页面加载 / 路由进入 | 标题 fade-in-left，描述 fade-in | 500-800ms |
| **CTA 区块** | 滚动揭示 | fade-up | 800ms |

**不做的事**：不做筛选面板、Apply Filter、筛选结果切换、分页切换等筛选型交互。产品卡片也不做密集滚动揭示，避免过度动画影响能力展示型页面节奏。

### 5.3 产品详情页 (/products/[slug])

参考 rticoutdoors.com 的叙事式布局动画节奏：

| 元素 | 触发 | 效果 | 时长 |
|------|------|------|------|
| **Section 1 — 图片画廊** | 缩略图点击 | 主图 opacity 切换（当前淡出 → 新图淡入） | Normal 500ms |
| **Section 1 — 产品信息** | 页面加载 | 产品名 fade-in 500ms，卖点列表 fade-up 交错入场 | 500-800ms |
| **Section 2 — 特性图标卡片** | 滚动揭示 | 4 张卡片 fade-up 交错入场，每张延迟 100ms | 800ms + stagger |
| **Section 3 — 叙事段落（左图右文）** | 滚动揭示 | 图片 fade-in 500ms，文字 fade-in-left 800ms | 500-800ms |
| **Section 3 — 叙事段落（右图左文）** | 滚动揭示 | 图片 fade-in 500ms，文字 fade-in（从右） 800ms | 500-800ms |
| **Section 3 — 嵌入视频** | 滚动进入视口 | autoplay 开始播放（Intersection Observer 控制） | — |
| **Section 4 — 规格表** | 滚动揭示 | 整块 fade-up | 800ms |
| **Section 5 — 相关产品** | 滚动揭示 | 卡片 fade-up 交错入场 | 800ms + stagger |
| **Send Inquiry 按钮** | hover | 同主按钮效果（box-shadow inset 填充） | Fast 300ms |

**叙事段落的关键**：左右交替布局中，文字的入场方向也交替（左侧 fade-in-left / 右侧 fade-in-right），配合图片的简单 fade-in，创造出「翻页阅读」的节奏感。

## 六、无障碍适配

对于设置了「减少动画」偏好的用户，全局禁用所有动画：

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

这不是可选项，是必须实现的。

## 七、性能约束

| 约束 | 标准 |
|------|------|
| 同时运行的动画 | 同一视口内不超过 6 个 |
| 可动画属性 | 仅 `opacity`、`transform`、`box-shadow` |
| 禁止动画的属性 | `width`、`height`、`margin`、`padding`、`top/left` |
| 首屏动画总时长 | 1.2 秒内全部完成（Hero 文字入场） |
| 滚动揭示触发频率 | 使用 `threshold: 0.2`，每个元素只触发一次 |
| 移动端特殊处理 | Hero 视频降级为静态图、位移距离减半、stagger 间隔减半 |

## 八、动画清单总览

按类型汇总，方便开发时逐项实现和核对：

### CSS Transitions（通过 Tailwind class 实现）

| 编号 | 效果 | Tailwind class 参考 | 应用位置 |
|------|------|---------------------|---------|
| T1 | 主按钮 hover（box-shadow inset 填充） | 自定义 class | 全站主按钮 |
| T2 | 次按钮 hover（背景填充） | `transition-all duration-300` | 全站次按钮 |
| T3 | 链接下划线显现 | `after:transition-all after:duration-300` | 全站文字链接 |
| T4 | 导航链接颜色 | `transition-colors duration-150` | Header |
| T5 | 下拉菜单展开 | `transition-all duration-300` | Header |
| T6 | 页脚图标填充 | 自定义 class（box-shadow inset） | Footer |
| T7 | 产品卡片上浮 | `transition-all duration-300 hover:-translate-y-1` | 产品列表 |
| T8 | 卡片图片放大 | `transition-transform duration-500 hover:scale-105` | 产品列表、Blog、首页分类 |
| T9 | 画廊图片切换 | `transition-opacity duration-500` | 产品详情 |
| T10 | 表单 focus | `transition-colors duration-150` | Contact、询盘表单 |
| T11 | 复选框 toggle | `transition-all duration-150` | 筛选面板 |
| T12 | 筛选面板收展 | `transition-all duration-300` | 产品列表移动端 |

### Keyframe Animations（通过 Intersection Observer 触发）

| 编号 | 效果 | 时长 | 应用位置 |
|------|------|------|---------|
| K1 | fade-up（上移淡入） | 800ms | 默认滚动揭示效果，大量使用 |
| K2 | fade-in（纯淡入） | 500ms | 图片、徽章、简单元素 |
| K3 | fade-in-left（左移淡入） | 800ms | 强调标题、叙事段落左侧 |
| K4 | fade-in-right（右移淡入） | 800ms | 叙事段落右侧 |
| K5 | scale-in（缩放淡入） | 300ms | 时间轴节点、WhatsApp 按钮 |
| K6 | 交错入场 stagger | 每项 +100ms 延迟 | 卡片组、列表组 |
| K7 | 数字滚动 | 2000ms | 首页数据、About Us 数据 |
| K8 | Hero 文字序列入场 | 800ms + 300ms 延迟链 | 首页 Hero |
